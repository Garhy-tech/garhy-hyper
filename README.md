# GARHY | HYPER

A full-stack e-commerce storefront and admin dashboard built on **TanStack Start**
(React 19 SSR). Bilingual (Arabic / English, with RTL support), backed by Supabase
for auth and data.

## Tech stack

| Concern        | Choice                                                  |
| -------------- | ------------------------------------------------------- |
| Framework      | TanStack Start + TanStack Router (file-based routing)   |
| UI runtime     | React 19 (server-side rendering)                        |
| Server bundle  | Nitro (`node-server` preset â†’ `.output/server/index.mjs`) |
| Data / auth    | Supabase (`@supabase/ssr`, PostgREST)                   |
| Data fetching  | TanStack Query + TanStack Start server functions        |
| Styling        | Tailwind CSS v4 (`@tailwindcss/vite`) + Lightning CSS   |
| i18n           | i18next / react-i18next (AR + EN, RTL aware)            |
| Animation      | Framer Motion                                           |
| Icons          | lucide-react                                            |

## Getting started

```bash
# 1. Install dependencies
npm install

# 2. Configure environment
cp .env.example .env
#   then fill in your Supabase values (see docs/ENVIRONMENT.md)

# 3. Start the dev server (http://0.0.0.0:5000)
npm run dev
```

## Scripts

| Script              | Description                                            |
| ------------------- | ------------------------------------------------------ |
| `npm run dev`       | Start the Vite dev server on `0.0.0.0:5000`            |
| `npm run build`     | Production build (emits `.output/server/index.mjs`)    |
| `npm run start`     | Run the built production server (`node .output/...`)   |
| `npm run preview`   | Preview the production build locally                   |
| `npm run lint`      | ESLint over `src`, `vite.config.ts`, `eslint.config.js` |
| `npm run format`    | Prettier write over the same paths                     |

Type-check with `npx tsc --noEmit`.

## Database & empty states

The app reads catalog data from Supabase via PostgREST. If the database schema
has not been migrated or tables are empty, the data layer **fails gracefully** â€”
PostgREST 404/406 responses are caught and surfaced as empty arrays, so the UI
renders localized empty states and loading skeletons rather than crashing.
Applying migrations and seeding data is handled separately (outside this app).

## Deployment

Build, then run the standalone Node server:

```bash
npm run build
npm run start   # node .output/server/index.mjs
```

Provide the four Supabase environment variables in the production environment.
See [docs/ENVIRONMENT.md](docs/ENVIRONMENT.md).
